# Documentation for `w90unk2unk.f90`

## Overview

`w90unk2unk.f90` is a Fortran utility program designed to convert Wannier90 `UNK` files from their default binary (unformatted) representation to a human-readable text (formatted) representation, or vice-versa (though the current version primarily focuses on unformatted to formatted). `UNK` files store the Bloch wavefunctions on a real-space grid at each k-point, which are used by Wannier90.

This utility is particularly useful for:
*   Inspecting the contents of `UNK` files.
*   Transferring `UNK` files between machines with different binary architectures where direct binary transfer might lead to compatibility issues.
*   Debugging or comparing wavefunction data.

The script handles both collinear (`UNKxxxxx.1`, `UNKxxxxx.2`) and non-collinear (`UNKxxxxx.NC`) `UNK` files.

## Key Components

The Fortran program `w90unk2unk.f90` consists of a single main program block.

*   **Main Program `w90unk2unk`**:
    *   **Argument Parsing**: Reads command-line arguments. The first argument determines the type of `UNK` file (collinear or non-collinear), and subsequent arguments are the paths to the `UNK` files to be converted.
        *   `soc` (integer): Set to `0` for collinear files, `1` for non-collinear files.
    *   **File I/O**:
        *   Opens the input unformatted `UNK` file (e.g., `UNK00001.1` or `UNK00001.NC`).
        *   Reads the header information: `ngx`, `ngy`, `ngz` (grid dimensions), `nk` (k-point index), `nbnd` (number of bands).
        *   Allocates an array `r_wvfn_nc` to store the wavefunction coefficients. For non-collinear files, this array has an additional dimension for the two spinor components.
        *   Reads the wavefunction coefficients band by band. For non-collinear files, it reads the two spinor components sequentially for each band.
        *   Closes the input file.
        *   Opens an output file, appending `.formatted` to the original filename (e.g., `UNK00001.1.formatted`).
        *   Writes the header information to the output file in formatted text.
        *   Writes the wavefunction coefficients to the output file in formatted text, using the `2ES20.10` format descriptor.
        *   Closes the output file.
    *   **Memory Management**: Allocates and deallocates the `r_wvfn_nc` array for each file processed.
    *   **Looping**: Iterates over all file paths provided as command-line arguments after the first `soc` argument.

## Important Variables/Constants

Within the `w90unk2unk.f90` code:

*   **`ngx, ngy, ngz`** (Integer): Dimensions of the real-space grid.
*   **`nk`** (Integer): The k-point index to which the `UNK` file corresponds.
*   **`nbnd`** (Integer): The number of bands in the `UNK` file.
*   **`soc`** (Integer, from command line): Flag indicating the file type:
    *   `0`: Collinear (`UNKxxxxx.1` or `UNKxxxxx.2`). The third dimension of `r_wvfn_nc` is 1.
    *   `1`: Non-collinear (`UNKxxxxx.NC`). The third dimension of `r_wvfn_nc` is 2 (for the two spinor components).
*   **`r_wvfn_nc(:,:,:)`** (Complex, Allocatable): Array storing the wavefunction coefficients. Dimensions are `(ngx*ngy*ngz, nbnd, num_spinor_components)`, where `num_spinor_components` is 1 for collinear and 2 for non-collinear.
*   **File Units**:
    *   Unit `1`: Used for reading the input unformatted `UNK` file.
    *   Unit `2`: Used for writing the output formatted `UNK` file.

## Usage Examples

1.  **Compilation**:
    Compile the Fortran source code using a Fortran compiler (e.g., gfortran):
    ```bash
    gfortran -o w90unk2unk w90unk2unk.f90
    ```
    This will create an executable file named `w90unk2unk`.

2.  **Execution**:
    Run the compiled program from the command line.

    *   **For collinear spin/spin-degenerate `UNK` files (e.g., `UNK00001.1`):**
        ```bash
        ./w90unk2unk 0 UNK00001.1
        ```
        This will generate a new formatted file named `UNK00001.1.formatted`. You can list multiple files:
        ```bash
        ./w90unk2unk 0 UNK*.1
        ```

    *   **For non-collinear `UNK` files (e.g., `UNK00001.NC`):**
        ```bash
        ./w90unk2unk 1 UNK00001.NC
        ```
        This will generate a new formatted file named `UNK00001.NC.formatted`. You can list multiple files:
        ```bash
        ./w90unk2unk 1 UNK*.NC
        ```

    The first argument (`0` or `1`) specifies whether the files are collinear or non-collinear, respectively. Subsequent arguments are the paths to the `UNK` files.

## Dependencies and Interactions

*   **Fortran Compiler**: A Fortran compiler (like `gfortran`) is required to compile the `w90unk2unk.f90` source code.
*   **Input Files**: Expects `UNK` files generated by Wannier90 or a VASP calculation with the VASP2Wannier90 interface enabled (e.g., `LWRITE_UNK = .TRUE.` in VASP's INCAR).
*   **Output Files**: Generates new files with the `.formatted` suffix, containing the text representation of the input `UNK` files. It does not modify the original files.

The script is standalone and does not depend on other parts of the `VASP2WAN90_v2_fix` project for its execution, though it processes files generated by VASP/Wannier90 systems that might use the accompanying patch.
